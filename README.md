# ⛈️ ARIA v3.0: 降雨動態地圖與災害模擬分析系統
**Week 5 Lab: Dynamic Mapping & Time-Machine Simulation**

---

## 📖 專案簡介
本專案為空間資訊系統課程之實驗成果。核心目標是建立一個能夠整合「即時氣象資料」與「歷史災害情境」的防災決策支援系統。透過 Python 串接氣象署 (CWA) API，並結合 GeoPandas 空間分析，評估避難所在極端降雨下的安全風險。

---

## 🚀 核心功能
* **即時 API 串接**：自動獲取並解析中央氣象署 (CWA) 降雨觀測資料。
* **時光機模擬 (Simulation)**：重現 2025 年「鳳凰颱風 (Typhoon Fung-wong)」蘇澳 130.5mm/hr 之極端降雨情境。
* **空間風險評估**：利用 **Buffer 分析** 與 **Spatial Join** 自動篩選受威脅避難所。
* **互動式視覺化**：產出包含熱度圖 (HeatMap) 與動態風險標記 (Exclamation Markers) 的 Folium 地圖。

## 📊 實驗心得與問答 (Lab Questions)

### 1. 座標系統 (CRS) 的重要性
若忘記轉換 CRS，程式會因為單位不統一（**度 vs 公尺**）導致緩衝區範圍失控（例如 5 公里變成 5000 度），或直接觸發 `ValueError` 導致程式中斷，無法正確判斷避難所是否落在降雨威脅區內。

### 2. CWA 為何使用 -998？
`-998` 是為了相容較舊的軟體格式。在現代資料分析中，必須將其轉換為 `NaN` 或 `0` 才能確保程式順利運行，並避免統計計算（如平均值）產生偏差。

### 3. 優先撤離決策
在鳳凰颱風情境中，建議優先撤離受蘇澳 **130.5mm** 雨帶覆蓋且 **dynamic_risk = CRITICAL** 的避難所。秉持「雨量決定威脅速度，地形決定致災強度」的決策原則。

### 4. 實驗挑戰與解決
最大的挑戰在於座標系統轉換（經緯度與公尺的對齊）。此外，模擬颱風在宜蘭而避難所在花蓮，距離太遠（約 70km）導致預設緩衝區抓不到資料，透過將影響半徑調整為 **80 公里**，成功驗證了空間風險連結的邏輯。

1. 請先參考 `.env.example` 建立 `.env` 檔案並填入您的 `CWA_API_KEY`。

3. 執行 `Week5_Student.ipynb` 即可在 `output/` 資料夾產出分析地圖。
